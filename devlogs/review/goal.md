# AudioNotes 项目总功能目标

## 项目定位

AudioNotes 是一款**离线优先**的语音转待办事项移动应用，核心价值在于：用户无需网络、无需手动输入，通过语音即可快速创建结构化的待办任务，并借助分类、标签、优先级等体系进行高效管理。

---

## 一、核心功能目标

### 1. 离线语音识别（ASR）
- 使用 Vosk ASR 引擎实现完全离线的语音识别，无需网络依赖
- 实时流式转录：录音时即时显示部分识别结果
- 语音活动检测（VAD）：基于停顿自动断句，将语音分段为独立条目
- 支持多模型管理：下载、切换不同语言/精度的 Vosk 模型（中文小模型/大模型）
- 识别结果带置信度评分，辅助用户判断识别质量
- 长音频自动分片识别（AudioChunker），突破单次识别时长限制

### 2. 录音与音频管理
- 通过原生平台通道（MethodChannel）进行高质量 PCM16 音频录制
- 录音状态实时反馈（idle / recording / recognizing / completed / failed）
- 每个 Todo 保存原始音频文件，支持回放对照
- 孤立音频文件自动清理（AudioFileCleanup）
- 重新录制（Re-record）：对已有 Todo 重新录音替换文本与音频

### 3. 待办事项（Todo）全生命周期管理
- **创建**：语音自动创建 / 手动创建
- **识别中状态**：录音结束后先插入占位条目（recognizing），后台完成识别后更新
- **编辑**：修改文本、优先级、分类、标签、截止日期、提醒时间
- **完成/取消完成**：切换完成状态，带视觉反馈（删除线）
- **软删除**：标记删除但保留数据，可恢复
- **拖拽排序**：手动排序模式下支持拖拽调整顺序
- **批量操作**：多选模式下的批量完成、删除、移动分类

---

## 二、组织与分类目标

### 4. 分类分组视图
- Todo 按分类（Category）自动分组展示
- 未分类条目归入"未分类"组
- 已完成条目可选聚合为独立的"已完成"组
- 分类支持折叠/展开
- 分类间拖拽排序，分类内 Todo 拖拽排序
- 分类自定义名称与颜色

### 5. 标签系统
- 多标签支持：一个 Todo 可关联多个标签
- 标签自定义名称与颜色
- 标签筛选与查看

### 6. 优先级体系
- 三级优先级：低 / 中 / 高
- 新建 Todo 时可设置默认优先级
- 按优先级排序

### 7. 排序系统
- 二级排序：分组级别排序 + 组内排序
- 排序维度：手动 / 创建时间 / 截止日期 / 优先级
- 排序方向：升序 / 降序

---

## 三、提醒与通知目标

### 8. 本地提醒
- 为 Todo 设置提醒时间
- 本地通知推送（flutter_local_notifications）
- 重复提醒：支持每日/每周重复
- 提醒与 Todo 一一关联，存储于 reminders 表
- 应用启动时自动同步未触发的提醒
- Android 精确闹钟权限适配

---

## 四、个性化与设置目标

### 9. 主题与外观
- 浅色/深色/跟随系统/自定义颜色主题
- 自定义主色调
- 字体大小调节：小/中/大/自定义/跟随系统

### 10. 应用设置
- 默认优先级设置
- 是否聚合已完成 Todo
- 是否自动移除识别文本末尾句号
- ASR 模型选择与自动选择
- 应用语言切换（中文/英文）

### 11. 国际化（i18n）
- 支持中文（zh_CN）与英文（en）
- 使用 flutter_i18n 进行运行时翻译加载
- 提醒通知文本也随语言设置变化

---

## 五、数据与存储目标

### 12. 持久化存储
- SQLite 数据库存储所有结构化数据
- 数据库版本迁移（当前 v5），支持平滑升级
- 5 张核心表：todo_item / categories / tags / todo_tags / reminders
- 音频文件本地存储（WAV 格式）
- 设置通过 SharedPreferences 持久化

### 13. 数据完整性
- 外键约束与级联删除
- 软删除机制保留可恢复性
- 置信度跟踪辅助质量评估
- 原始转录文本保留（rawTranscript）

---

## 六、用户体验目标

### 14. 交互体验
- 流畅的动画效果（flutter_animate + 自定义 Motion 工具）
- 录音浮层实时显示识别状态
- 空状态引导新用户
- 拖拽排序触觉反馈
- 批量操作浮动工具栏

### 15. 跨平台支持
- Android（Kotlin 原生插件）
- iOS（Swift 原生插件）
- 理论支持 Web / Windows / Linux / macOS（UI 层），但原生 ASR 仅限移动端

---

## 架构目标

- **Clean Architecture**：Presentation → Domain → Data 三层分离
- **Riverpod** 状态管理：响应式、可测试、依赖注入
- **Repository 模式**：数据层抽象，便于替换存储实现
- **平台通道**：Dart 与原生代码通过 MethodChannel 通信
- **关注点分离**：录音（RecorderService）与识别（RecognitionService）解耦
