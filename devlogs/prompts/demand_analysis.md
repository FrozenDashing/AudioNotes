**用户故事**  
- 作为忙碌用户，我可以一键录音并立即看到转写结果，以便快速记录想法。  
- 作为健忘用户，我希望所有记录按录入时间排列，保留原始顺序。  
- 作为老年用户，我希望界面极简、操作少、字体大。

**功能清单**

6. **基础 CRUD**：标记完成/未完成、单条删除、批量删除、清空已完成。  
7. **本地数据库**：使用 SQLite/Room/CoreData 存储条目、状态、时间戳、音频引用。

- 协助撰写 VAD 调参建议与离线模型选型对比。  
**AI 预估效率提升**：约 20–30%。

### 阶段 2：功能增强（目标：提升可用性与日常使用体验）
**目标**  
- 增加提醒、悬浮快速入口与列表个性化，提升日常使用黏性与便捷性。

**范围（不包含）**  
- 云端多设备同步、复杂权限管理、商业化功能。

**用户故事**  
- 作为用户，我可以为某条待办设置提醒，到点收到弹窗与语音提示。  
- 作为经常外出用户，我可以在任意界面通过悬浮按钮快速录音。  
- 作为偏好简洁的用户，我可以隐藏已完成项并切换暗黑模式。

**功能清单**  
1. **提醒与闹钟**：单条设置截止时间、一次性提醒、每日/每周重复规则。  
2. **本地通知实现**：使用平台通知中心（Android AlarmManager/WorkManager、iOS UNUserNotificationCenter）。  
3. **悬浮录音按钮**：实现系统悬浮窗（Android）或快捷方式（iOS Shortcuts/Widget）。  
4. **列表个性化**：极简模式切换、折叠/隐藏已完成、主题切换（暗黑/浅色）。  
5. **重录替换音频**：支持对单条进行重新录音并替换转写。  

**验收标准**  
- 提醒按设定时间触发，重复规则正确执行。  
- 悬浮按钮在允许权限下可从桌面/其他应用快速唤起录音。  
- 列表个性化设置在重启后保持。

**界面/流程要点**  
- 提醒设置入口在条目详情页或长按菜单。  
- 悬浮按钮样式可配置（小/大、透明度）。  
- 设置页提供主题与折叠选项。

**测试要点**  
- 各类重复规则的触发测试（跨日、跨周）。  
- 悬浮按钮在不同系统版本与权限状态下的行为。  
- 通知在锁屏/Doze 模式下的可靠性。

**风险与缓解**  
- **悬浮窗权限限制（尤其 iOS）**：Android 实现悬浮窗，iOS 提供 Widget/快捷方式替代。  
- **通知被系统限制**：提供用户引导开启自启动/通知权限。

**交付物**  
- 增强功能版本安装包；提醒与悬浮功能的集成文档；测试用例与结果。  

**估时（团队：1 移动、1 QA、1 后端/若需本地服务）**  
- **3–4 周**。  
**AI 协助点**  
- 生成通知与重复规则实现代码模板。  
- 提供跨平台悬浮/Widget 实现方案与示例代码。  
- 自动生成 UI 测试脚本与用例。

### 阶段 3：进阶迭代（目标：数据安全与扩展功能）
**目标**  
- 实现云端备份/迁移、可选分类/标签，提升数据安全与长期使用价值。

**范围（不包含）**  
- 多语言/方言支持（本阶段不考虑）、复杂协作功能。

**用户故事**  
- 作为换机用户，我希望一键迁移所有待办到新设备。  
- 作为重度用户，我希望用标签/分类快速筛选事项。

**功能清单**  
1. **云端备份与同步**：账号体系（可选匿名/手机号/第三方登录），数据加密传输与存储。  
2. **冲突解决策略**：基于时间戳的合并策略与用户手动冲突解决。  
3. **分类与标签**：预设分类（生活/工作/私事/预约）与自定义标签，按标签筛选。  
4. **数据导入导出**：支持本地导出（JSON/备份文件）与导入。  

**验收标准**  
- 同一账号在两台设备间能完成数据同步且无数据丢失。  
- 标签/分类筛选响应时间 < 200ms（本地查询）。  
- 备份与恢复流程在网络中断后能正确重试并完成。

**界面/流程要点**  
- 账号与同步设置页，提供备份状态与最后同步时间。  
- 冲突提示界面简洁明了，提供“保留本地/保留云端/合并”选项。

**测试要点**  
- 大量数据同步测试（上千条）与冲突场景测试。  
- 加密传输与恢复完整性校验。  

**风险与缓解**  
- **隐私合规与数据泄露风险**：采用端到端或传输层加密，明确隐私条款并最小化云端存储敏感信息。  
- **同步冲突复杂**：优先采用简单明确的时间戳优先策略并提供用户可控选项。

**交付物**  
- 支持云备份/迁移的版本；同步架构文档；隐私与数据处理说明。  

**估时（团队：1 后端/云、1 移动、1 QA、安全顾问）**  
- **4–5 周**。  
**AI 协助点**  
- 生成 API 文档草稿、同步冲突处理伪代码。  
- 提供加密实现建议与合规性检查清单。  
- 自动化同步测试脚本生成。

### 阶段 4：优化与发布（目标：稳定、性能与上线）
**目标**  
- 全面修复 BUG、性能优化（语音延迟、内存、启动时间）、完成上架准备与合规文档。

**范围**  
- 包括但不限于性能调优、无障碍优化、隐私合规、应用商店上架材料。

**功能清单 / 工作项**  
1. **性能优化**：降低语音转写延迟（目标 <1s 响应起始）、内存与电量优化。  
2. **稳定性与兼容性测试**：覆盖主流机型与系统版本。  
3. **无障碍支持**：大字号、语音提示、可访问性标签。  
4. **隐私与法律合规**：隐私政策、用户协议、数据处理说明。  
5. **上架准备**：应用商店截图、描述、测试账号、版本发布流程。  

**验收标准**  
- 关键路径（录音→转写→保存）平均延迟满足目标。  
- Crash rate < 0.5%（或按团队目标）。  
- 无障碍检查通过主要辅助工具测试。

**测试要点**  
- 自动化回归测试、性能基准测试、压力测试。  
- 上架审核前的合规自检。

**交付物**  
- 最终发布版本、性能测试报告、上架材料、用户帮助文档与隐私政策。

**估时（团队：全体参与，含产品/PM）**  
- **2–3 周**。  
**AI 协助点**  
- 生成性能测试脚本、发布文档、隐私政策草稿与常见问题文档。  
- 协助生成上架所需的文案与截图文案建议。
