# 动画统一风格 Draft

首先,在设置,外观设置中添加减少动效开关,如果打开就是没有以下额外动画,关闭就是有动画
## 目标

为 AudioNotes 建立一套统一、克制、可感知的动态语言，让界面在所有交互场景下都表现出一致的节奏、层级和反馈。

重点不是“更多动画”，而是让所有动画都遵循同一套规则：
- 同样的进入方式
- 同样的层级反馈
- 同样的时长分层
- 同样的缓动语气
- 同样的状态转换逻辑

这样用户在首页、分组、拖放、录音、设置和弹窗之间切换时，能持续感受到同一个产品，而不是多个局部效果的拼接。

## 现有元素的动态机会

从当前界面元素看，最适合统一动画风格的部分包括：
- 首页整体列表与分组卡片
- 分类组的展开/收起
- todo 条目的拖放、插入、移动
- 顶部工具区与浮动操作条
- 录音状态覆盖层与识别状态切换
- 弹窗、底部抽屉、设置页的进入/退出
- 完成状态、选中状态、空状态等反馈

这些部分都已经有明确的视觉层级，但动态反馈比较分散。统一动画策略可以把“可操作性”做得更强，同时减少界面切换时的突兀感。

## 建议的统一动态语言

### 1. 时间结构统一

建议把全局动画分成三档：
- 80ms 到 120ms：微反馈，例如按钮按下、图标切换、轻量 hover
- 160ms 到 220ms：常规界面过渡，例如卡片展开、列表项位置变化、状态切换
- 260ms 到 360ms：结构性变化，例如分类展开、底部面板、录音覆盖层

原则是：
- 同类交互使用同一时长
- 层级越大，动画越慢，但不要超过 360ms 的常态上限
- 退出通常比进入略快，避免界面“拖尾感”太重

### 2. 缓动曲线统一

建议形成固定的曲线语义：
- 轻反馈：`easeOut`
- 位置变化：`easeInOut` 或接近自然物理感的缓动
- 强调进入：前快后稳，避免过度弹跳
- 弹窗和抽屉：轻微减速收尾，表现“浮层压住内容”的感觉

不建议在同一产品中同时使用太多风格差异明显的曲线，尤其是“弹性过强”和“线性生硬”混用，会让界面语气不统一。

### 3. 空间规则统一

所有卡片、列表、抽屉和浮层尽量沿着相似的空间语法移动：
- 从原位置轻微位移进入
- 从当前位置轻微缩放或透明度变化退出
- 避免无来源的突然跳变
- 避免大跨度位移配合过强缩放

这样用户会感觉元素一直“属于”当前界面，而不是突然出现或消失。

## 按元素的建议

### 首页与列表区

首页是整个产品的主节奏入口，建议作为动画风格的基准页。

可做的统一化处理：
- 列表首次加载时采用轻微渐入和顺序出现
- 分组卡片按顺序做微弱 stagger，形成层次
- 空状态、加载状态、加载失败状态都用同一套过渡语气
- 滚动过程中保持内容稳定，不做过多跟随动画

建议首页承担“整体节奏”的定义，不要太花，但要稳定、连续。

### 分类组卡片

分类组是最适合建立品牌动态语言的组件。

建议：
- 展开/收起使用高度变化配合轻微透明度和内容位移
- 组头部保持固定节奏，避免展开时头部也大幅晃动
- 左侧色条或强调色可用极轻微呼吸感，但不要持续闪动
- 分组间拖动时，空位、目标位和悬浮卡片要有明确反馈层级

分组动画应当体现“容器感”，即内容是从组里长出来，而不是单独飞出来。

### todo 条目

todo 项是最频繁交互的对象，动画应该最克制但最明确。

建议：
- 添加任务：从列表底部或目标组中轻微插入，不要突然跳到位
- 删除任务：优先用收缩 + 淡出，而不是直接消失
- 完成状态：可以用颜色、透明度、文字装饰的协同变化
- 选中状态：只需要轻量高亮，不要和完成状态抢戏
- 任务顺序变化：拖动时悬浮卡片与占位位移要足够清晰

todo 项的动画重点是“反馈准确”，而不是“动作丰富”。

### 拖放系统

拖放是最容易暴露风格不统一的地方，建议单独建立规则。

建议统一以下行为：
- 拖起时卡片轻微升高并带一点阴影
- 其他项轻微让位，目标位置明确显示
- 释放后过渡回列表位置的时长要短于拖动中的视觉反馈
- 跨组移动时，目标组应有接纳感，而不是仅仅换位置

拖放动画是“空间归属感”的核心，建议和首页列表重排统一处理。

### 工具条与快捷按钮

浮动工具条、顶部按钮、筛选入口应保持轻反馈：
- 点击缩放 0.96 到 0.98 的轻压感
- 出现和隐藏以淡入淡出为主，辅以轻微位移
- 避免过多弹跳或旋转，保持工具属性

这些组件不应该抢夺主内容的注意力，只负责确认“系统可响应”。

### 录音与识别状态

录音覆盖层、识别中状态、完成提示属于高语义状态，建议更明确一些。

建议：
- 录音开始时有较清晰的层级抬升
- 识别中状态保持轻微呼吸或进度感，但不要过度炫技
- 识别成功/失败应使用一致的进入和退出逻辑
- 录音状态的出现/消失最好与底层列表分离，表现为覆盖层，而非列表本身剧烈变化

这部分动画应当帮助用户理解“系统正在工作”，而不是装饰。

### 弹窗与底部抽屉

设置、排序、确认删除等浮层建议统一使用：
- 自下而上或中心缩放的一致进入方式
- 内容区和背景遮罩同步变化
- 关闭时先淡出内容，再释放背景

这样所有浮层都像是来自同一个层级系统。

## 推荐的统一风格关键词

如果要给整个产品的动画风格下定义，建议是：
- 温和
- 清晰
- 有节奏
- 少量弹性
- 轻量空间感
- 不喧宾夺主

它应该像一个“稳重但有温度”的工具型产品，而不是娱乐型界面。

## 落地优先级

建议优先做下面几块：

### 第一优先级
- 首页分组与 todo 列表重排
- 分类展开/收起
- 拖放悬浮卡片与目标占位

这三处决定用户是否能感知整体动态风格。

### 第二优先级
- 录音覆盖层与识别状态
- 底部抽屉和设置弹窗
- 按钮和工具条反馈

这些部分决定交互是否连续。

### 第三优先级
- 完成状态变化
- 空状态加载态
- 细小图标与辅助反馈

这些属于增强层，不影响主体验但能提升精致度。

## 需要避免的问题

- 不同页面各自使用不同缓动和时长
- 同类反馈同时出现位移、缩放、旋转、弹性过强
- 拖放动画太慢，导致用户误以为操作失败
- 列表刷新和本地状态变化的动画不一致
- 录音、删除、完成等状态变化缺乏统一节奏

## 建议的实现方式

如果后续进入实现阶段，建议不要把动画散落在各个 widget 里临时加。
更好的方式是先抽出一组统一的动画 token：
- duration token
- curve token
- entrance token
- exit token
- emphasis token

然后让首页、分组、卡片、浮层都复用这些 token。

这样后续新增功能时，也能自然继承同一种动态语言。

## 结论

AudioNotes 最适合的不是强装饰动画，而是一套“安静但一致”的动态系统。
它的目标是：
- 让层级更清楚
- 让操作更可感知
- 让状态变化更可信
- 让整个产品看起来是一个完整系统，而不是功能拼装

如果按这个方向推进，动画会成为产品气质的一部分，而不是额外负担。
